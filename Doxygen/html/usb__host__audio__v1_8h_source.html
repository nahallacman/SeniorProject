2 
3  USB Host Audio v1.0 Client Driver Header File
4 
5 Description:
6  This is the header file for a USB Embedded Host that is using the Audio
7  Class v1.0
8 
9  This file should be included with usb_host.h to provide the USB hardware
10  interface. It must be included after the application-specific usb_config.h
11  file and after the USB Embedded Host header file usb_host.h, as definitions
12  in those files are required for proper compilation.
13 
14  To interface with usb_host.c, the routine USBHostAudioV1Initialize() should
15  be specified as the Initialize() function, and USBHostAudioV1EventHandler()
16  should be specified as the EventHandler() function in the
17  usbClientDrvTable[] array declared in usb_config.h.
18 
19  This driver can be configured to use transfer events from usb_host.c.
20  Transfer events require more RAM and ROM than polling, but it cuts down or
21  even eliminates the required polling of the various USBxxxTasks functions.
22  For this class, USBHostMSDTasks() is compiled out if transfer events from
23  usb_host.c are used. However, USBHostTasks() still must be called to
24  provide attach, enumeration, and detach services. If transfer events from
25  usb_host.c are going to be used, USB_ENABLE_TRANSFER_EVENT should be
26  defined. If transfer status is going to be polled,
27  USB_ENABLE_TRANSFER_EVENT should not be defined.
28 
29  This driver can also be configured to provide mass storage transfer events
30  to the next layer. Generating these events requires a small amount of
31  extra ROM, but no extra RAM. The layer above this driver must be
32  configured to receive and respond to the events. If mass storage transfer
33  events are going to be sent to the next layer,
34  USB_MSD_ENABLE_TRANSFER_EVENT should be defined. If mass storage transfer
35  status is going to be polled, USB_MSD_ENABLE_TRANSFER_EVENT should not be
36  defined.
37 
38  Since audio support is performed with both isochronous and interrupt
39  transfers, USB_SUPPORT_INTERRUPT_TRANSFERS and
40  USB_SUPPORT_ISOCHRONOUS_TRANSFERS must be defined.
41 
42 Summary:
43  This is the header file for a USB Embedded Host that is using the Audio
44  Class v1.0.

151  // No event occured (NULL event).
152 #define EVENT_AUDIO_NONE EVENT_AUDIO_BASE + EVENT_AUDIO_OFFSET + 0
153  // An audio device has attached. The returned data pointer points to a
154  // USB_AUDIO_V1_DEVICE_ID structure.
155 #define EVENT_AUDIO_ATTACH EVENT_AUDIO_BASE + EVENT_AUDIO_OFFSET + 1
156  // An audio device has detached. The returned data pointer points to a
157  // byte with the previous address of the detached device.
158 #define EVENT_AUDIO_DETACH EVENT_AUDIO_BASE + EVENT_AUDIO_OFFSET + 2
159  // An audio stream data packet has been received. The returned data
160  // pointer points to a HOST_TRANSFER_DATA structure, with information about
161  // the most recent transfer. One event will be returned for each transfer,
162  // so the application will know how much data was actually received in
163  // each transfer. If there was a bus error, both the returned data pointer
164  // and the size will be zero.
165 #define EVENT_AUDIO_STREAM_RECEIVED EVENT_AUDIO_BASE + EVENT_AUDIO_OFFSET + 3
166  // This event is returned after the sampling frequency is set via
167  // USBHostAudioV1SetSamplingFrequency(). The returned data pointer points
168  // to a HOST_TRANSFER_DATA structure, with the error code for this request.
169 #define EVENT_AUDIO_FREQUENCY_SET EVENT_AUDIO_BASE + EVENT_AUDIO_OFFSET + 4
170  // This event is returned after the full or zero bandwidth interface has
171  // been set. The returned data pointer is NULL, but the size is the

306 /****************************************************************************
307  Function:
308  BYTE USBHostAudioV1SetSamplingFrequency( BYTE deviceAddress, BYTE *frequency )
309 
310  Summary:
311  This function sets the sampling frequency for the device.
312 
313  Description:
314  This function sets the sampling frequency for the device. If the exact
315  frequency is not supported by the device, the device will round it to the
316  closest supported value.
317 
318  IMPORTANT: If the request is initiated successfully, the frequency value
319  must remain valid until the EVENT_AUDIO_FREQUENCY_SET event is received.
320  Therefore, this value cannot be a local (stack) variable. The application
321  can either use a global variable for this value, or it can use the
322  function USBHostAudioV1SupportedFrequencies() to obtain a pointer to the
323  number and list of supported frequencies, and pass a pointer to the desired
324  frequency in this list.
325 
326  Precondition:
327  None
328 
329  Parameters:
330  BYTE deviceAddress - Device address
331  BYTE *frequency - Pointer to three bytes that specify the desired
332  sampling frequency. NOTE: If the request is
333  initiated successfully, this location must
334  remain valid until the EVENT_AUDIO_FREQUENCY_SET
335  event is received.
336 
337  Return Values:
338  USB_SUCCESS - Request started successfully
339  Others - See USBHostIssueDeviceRequest() errors.
340 
341  Example:
342  <code>
343  BYTE numFrequencies;
344  BYTE *ptr;
345 
346  ptr = USBHostAudioV1SupportedFrequencies( deviceAddress );
347  if (ptr)
348  {
349  numFrequencies = *ptr;
350  ptr++;
351  if (numFrequencies == 0)
352  {
353  // Continuous sampling, minimum and maximum are specified.
354  DWORD minFrequency;
355  DWORD maxFrequency;
356 
357  minFrequency = *ptr + (*(ptr+1) << 8) + (*(ptr+2) << 16);
358  ptr += 3;
359  maxFrequency = *ptr + (*(ptr+1) << 8) + (*(ptr+2) << 16);
360  if ((minFrequency <= desiredFrequency) && (desiredFrequency <= maxFrequency))
361  {
362  rc = USBHostAudioV1SetSamplingFrequency( deviceAddress, &desiredFrequency );
363  }
364  else
365  {
366  // Desired frequency out of range
367  }
368  }
369  else
370  {
371  // Discrete sampling frequencies are specified.
372  DWORD frequency;
373 
374  while (numFrequencies)
375  {
376  frequency = *ptr + (*(ptr+1) << 8) + (*(ptr+2) << 16);
377  if (frequency == desiredFrequency)
378  {
379  rc = USBHostAudioV1SetSamplingFrequency( deviceAddress, ptr );
380  continue;
381  }
382  numFrequencies--;
383  ptr += 3;
384  }
385  if (numFrequencies == 0)
386  {
387  // Desired frequency not found.
388  }
389  }
390  }
391  </code>
392 
393  Remarks:
394  If a global variable is used to old the frequency, it can be declared as
395  a DWORD. Since PIC Microcontrollers are little endian machines, a
396  pointer to the DWORD can be used as the frequency parameter:
397  <code>
398  DWORD desiredFrequency = 44100; // Hertz
399 
400  rc = USBHostAudioV1SetSamplingFrequency( deviceAddress, (BYTE *)(&desiredFrequency) );
401  </code>
