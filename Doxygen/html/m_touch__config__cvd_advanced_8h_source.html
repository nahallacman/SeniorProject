71  #define CVD_SETTLING_DELAY 5 // <-- Adjusts the number of additional instruction cycles
72  // to add to the Chold&Csensor settling time.
73  // The larger this value is, the more susceptible to
74  // noise the application will be. However, there is a
75  // minimum amount of time required to get full
76  // sensitivity.

85  #define CVD_SWITCH_DELAY 0 // <-- Ideal value for noise immunity is 0.
86  // Determines how many NOPs will be added between
87  // when the sensor is set to an input and the ADC
88  // mux is pointed to the sensor. This value should
89  // ONLY be increased if the output driver is not
90  // disabling itself quickly enough. This will be
91  // noticeable on the second-sample of the waveform:
92  // the settling point will be slightly unstable and
93  // will be visibly coupling to VDD.
94  // The behavior is most-noticeable on the second scan
95  // due to P-type transistors being less ideal than
96  // N-type transistors.
97  // The behavior is most-common when Fosc is a high
98  // value (16MHz or greater).
99  // If you are seeing the behavior, the correct value
100  // for this option will vary based on Fosc and the
101  // microcontroller in question; but, in general,
102  // 1-3 NOPs for 16MHz, 3-5 for 32MHz.
