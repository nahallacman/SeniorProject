268  Summary:

270 
271  Description:
272  This routine takes an AES key and performs a key schedule to expand the key into a number of separate
273  set of round keys. These keys are commonly know as the Rijindael key schedule or a session key.
274 
275  Precondition:
276  None.
277 
278  Parameters:
279  round_keys - [out] Pointer to the output buffer that will contain the expanded short key (Rijindael) schedule/ session key. This is to be used in the
280  encryption and decryption routines. The round_keys buffer must be
281  word aligned for the target processor.
282  key - [in] The input key which can be 128, 192, or 256 bits in length.
283  key_size - [in] Specifies the key length in bytes. Valid options are\:
284  * AES_KEY_SIZE_128_BIT
285  * AES_KEY_SIZE_192_BIT
286  * AES_KEY_SIZE_256_BIT
287  The values 16, 24, and 32 may also be used instead of the above definitions.
288 
289  Returns:
290  None
291 
292  Example:
293  <code>
294  static const UINT8 AESKey128[] = { 0x95, 0xA8, 0xEE, 0x8E,

297  0x97, 0x97, 0x52, 0x8D
298  };
299  AES_ROUND_KEYS_128_BIT round_keys;
