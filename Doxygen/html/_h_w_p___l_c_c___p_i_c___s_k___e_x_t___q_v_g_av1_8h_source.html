316  Display Parameter Macros Descriptions:
317 
318  1. DISP_ORIENTATION - defines the display rotation with
319  respect to its native orientation. For example, if the display
320  has a resolution specifications that says 240x320 (QVGA), the
321  display is natively in portrait mode. If the application
322  uses the display in landscape mode (320x240), then the
323  orientation musts be defined as 90 or 180 degree rotation.
324  The Graphics Library will calculate the actual pixel location

2514 * uses for the viewable pixels. These two areas will exclusively be
2515 * used by the IPU operation to perform the decompression.
2516 * The use of the IPU cache areas assumes that the hardware used
2517 * is the PIC24FJ256DA210 Development Board where the EPMP CS1 is hooked
2518 * up to SRAM and EPMP CS2 is hooked up to a parallel flash memory.
2519 * Make sure that the buffer sizes are word aligned since IPU operates
2520 * only on WORD aligned addresses.
2521 * Users can derive the size of the two areas based on the output of the
2522 * "Graphics Resource Converter" tool used to generate the hex or C array
2523 * of the compressed images. Please refer to the "Graphics Resource
2524 * Converter" Help file for details.
2525 * If the compressed image is placed in Parallel Flash, then the
2526 * GFX_COMPRESSED_BUFFER_SIZE is not needed since the IPU module
2527 * can access the compressed data directly.
