199 /*********************************************************************
200 * Function: WORD TeTranslateMsg(void *pObj, GOL_MSG *pMsg)
201 *
202 * Overview: This function evaluates the message from a user if the
203 * message will affect the object or not. If the message
204 * is valid, the keys in the Text Entry object will be
205 * scanned to detect which key was pressed. If True, the
206 * corresponding text will be displayed, the ‘text’ will
207 * also be stored in the TeOutput parameter of the object.
208 *
209 * <TABLE>
210 * Translated Message Input Source Events Description
211 * ################## ############ ###### ###########
212 * TE_MSG_PRESS Touch Screen EVENT_PRESS, EVENT_MOVE If the event occurs and the x,y position falls in the face of one of the keys of the object while the key is unpressed.
213 * TE_MSG_RELEASED Touch Screen EVENT_MOVE If the event occurs and the x,y position falls outside the face of one of the keys of the object while the key is pressed.
214 * TE_MSG_RELEASED Touch Screen EVENT_RELEASE If the event occurs and the x,y position falls does not falls inside any of the faces of the keys of the object.
215 * TE_MSG_ADD_CHAR Touch Screen EVENT_RELEASE, EVENT_MOVE If the event occurs and the x,y position falls in the face of one of the keys of the object while the key is unpressed and the key is associated with no commands.
216 * TE_MSG_DELETE Touch Screen EVENT_RELEASE, EVENT_MOVE If the event occurs and the x,y position falls in the face of one of the keys of the object while the key is unpressed and the key is associated with delete command.
217 * TE_MSG_SPACE Touch Screen EVENT_RELEASE, EVENT_MOVE If the event occurs and the x,y position falls in the face of one of the keys of the object while the key is unpressed and the key is associated with space command.
218 * TE_MSG_ENTER Touch Screen EVENT_RELEASE, EVENT_MOVE If the event occurs and the x,y position falls in the face of one of the keys of the object while the key is unpressed and the key is associated with enter command.
219 * OBJ_MSG_INVALID Any Any If the message did not affect the object.
220 * </TABLE>
221 *
222 * PreCondition: none
223 *
224 * Input: pTe- The pointer to the object where the message will be
225 * evaluated to check if the message will affect the object.
226 * pMsg- Pointer to the message struct containing the message from
227 * the user interface.
228 *
229 * Output: Returns the translated message depending on the received GOL message:
230 * - TE_MSG_PRESS – A key is pressed
231 * - TE_MSG_RELEASED - A key was released (generic for keys with no commands or characters assigned)
232 * - TE_MSG_ADD_CHAR – A key was released with character assigned
233 * - TE_MSG_DELETE – A key was released with delete command assigned
234 * - TE_MSG_SPACE - A key was released with space command assigned
235 * - TE_MSG_ENTER - A key was released with enter command assigned
236 * - OBJ_MSG_INVALID – Text Entry is not affected
237 *
238 * Side Effects: none.
239 *
240 ********************************************************************/
241 WORD TeTranslateMsg(void *pObj, GOL_MSG *pMsg);

256 * TE_UPDATE_KEY,

259 * TE_UPDATE_KEY,

261 * TE_MSG_ENTER Touch Screen, Set TE_UPDATE_TEXT, User can define the use of this event in the message callback. Object will just update the key.
262 * TE_UPDATE_KEY,
263 * Clear TE_KEY_PRESSED
264 * TE_MSG_RELEASED Touch Screen, Clear TE_KEY_PRESSED A Key in the object will be redrawn in the unpressed state.
265 * Set Te_UPDATE_KEY
266 * TE_MSG_PRESSED Touch Screen, Set TE_KEY_PRESSED A Key in the object will be redrawn in the pressed state.
267 * TE_UPDATE_KEY
268 *
269 * </TABLE>
270 *
271 * PreCondition: none
272 *
273 * Input: translatedMsg - The translated message.

275 * pMsg - The pointer to the GOL message.

277 * Output: none

279 * Example:
280 * See BtnTranslateMsg() example.
281 *
282 * Side Effects: none
