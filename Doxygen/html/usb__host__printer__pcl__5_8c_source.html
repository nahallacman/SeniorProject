2  PCL 5 Printer Language Support
3 
4  Summary:
5  This file provides support for the PCL 5 printer language when using the
6  USB Embedded Host Printer Client Driver.
7 
8  Description:
9  This file provides support for the PCL 5 printer language when using the
10  USB Embedded Host Printer Client Driver.
11 
12  There are several versions of the PCL printer language. This file is
13  targetted to support PCL 5. Unfortunately, printer language support is not
14  always advertised correctly by the printer. Some printers advertise only
15  PCL 6 support when they also support PCL 5. Therefore, the default value
16  for the LANGUAGE_ID_STRING_PCL string used in the routine
17  USBHostPrinterLanguagePCL5IsSupported() is set such that the routine will
18  return TRUE if any PCL language support is advertised. It is highly
19  recommended to test the target application with the specific printer(s)
20  that will be utilized, and, if possible, populate the
21  usbPrinterSpecificLanguage[] array in usb_config.c via the configuration
22  tool to manually select the printer language and its functional support.
23 
24  Notes:
25  The PCL 5 coordinate origin is located at the top left corner of the paper.
26  The HP-GL/2 coordinate origin, however, is located at the bottom left corner
27  of the page. For consistency for the user, HP-GL/2 coordinate system is
28  adjusted to match the PCL coordinate system. This also matches the
29  coordinate system use by the Microchip Graphics library.
30 
31  The black and white bit image polarity is 0=white, 1=black, which is
32  reversed from the Microchip Graphics Library polarity. This driver will
33  automatically convert the image data to the required format, as long as the
34  image data is located in ROM (USB_PRINTER_TRANSFER_FROM_ROM) or it is
35  copied from a RAM buffer (USB_PRINTER_TRANSFER_COPY_DATA). If the data is
36  to be sent directly from its original RAM location, the data must already
37  be in the format required by the printer language.
38 
39  PCL 5 is not compatible with PCL 6; PCL 5 utilizes ASCII input, whereas
40  PCL 6 utilizes binary data. However, some printers that advertise support
41  for only PCL 5 do support PCL 6.
42 
43  PCL 3 printers utilize many of the PCL 5 commands. The following
44  limitations exist with PCL 3:
45  * PCL 3 does not support vector graphics
46  * PCL 3 does not support image printing in landscape mode. The
47  page print will fail.
48  * Items must be sent to the page in the order that they are to be
49  printed from top to bottom. The printer cannot return to a
50  higher position on the page.
51  * PCL 3 does not support the USB_PRINTER_EJECT_PAGE command. Use the
52  USB_PRINTER_JOB_STOP and USB_PRINTER_JOB_START commands instead.

334 /****************************************************************************
335  Function:
336  BYTE USBHostPrinterLanguagePCL5( BYTE address,
337  USB_PRINTER_COMMAND command, USB_DATA_POINTER data, DWORD size, BYTE transferFlags )
338 
339  Summary:
340  This function executes printer commands for a PCL 5 printer.
341 
342  Description:
343  This function executes printer commands for a PCL 5 printer. When
344  the application issues a printer command, the printer client driver
345  determines what language to use to communicate with the printer, and
346  transfers the command to that language support routine. As much as
347  possible, commands are designed to produce the same output regardless
348  of what printer language is used.
349 
350  Not all printer commands support data from both RAM and ROM. Unless
351  otherwise noted, the data pointer is assumed to point to RAM, regardless of
352  the value of transferFlags. Refer to the specific command to see if ROM
353  data is supported.
354 
355  Preconditions:
356  None
357 
358  Parameters:
359  BYTE address - Device's address on the bus
360  USB_PRINTER_COMMAND command - Command to execute. See the enumeration
361  USB_PRINTER_COMMAND for the list of
362  valid commands and their requirements.
363  USB_DATA_POINTER data - Pointer to the required data. Note that
364  the caller must set transferFlags
365  appropriately to indicate if the pointer is
366  a RAM pointer or a ROM pointer.
367  DWORD size - Size of the data. For some commands, this
368  parameter is used to hold the data itself.
369  BYTE transferFlags - Flags that indicate details about the
370  transfer operation. Refer to these flags
371  * USB_PRINTER_TRANSFER_COPY_DATA
372  * USB_PRINTER_TRANSFER_STATIC_DATA
373  * USB_PRINTER_TRANSFER_NOTIFY
374  * USB_PRINTER_TRANSFER_FROM_ROM
375  * USB_PRINTER_TRANSFER_FROM_RAM
376 
377  Return Values:
378  USB_PRINTER_SUCCESS - The command was executed successfully.
379  USB_PRINTER_UNKNOWN_DEVICE - A printer with the indicated address is not
380  attached
381  USB_PRINTER_TOO_MANY_DEVICES - The printer status array does not have
382  space for another printer.
383  USB_PRINTER_OUT_OF_MEMORY - Not enough available heap space to
384  execute the command.
385  other - See possible return codes from the
386  function USBHostPrinterWrite().
387 
388  Remarks:
389  When developing new commands, keep in mind that the function
390  USBHostPrinterCommandReady() will be used before calling this function to
391  see if there is space available in the output transfer queue.
392  USBHostPrinterCommandReady() will routine TRUE if a single space is
393  available in the output queue. Therefore, each command can generate only
394  one output transfer.
395 
396  Multiple printer languages may be used in a single application. The USB
397  Embedded Host Printer Client Driver will call the routine required for the
398  attached device.
399  ***************************************************************************/

401 BYTE USBHostPrinterLanguagePCL5( BYTE address,
402  USB_PRINTER_COMMAND command, USB_DATA_POINTER data, DWORD size, BYTE transferFlags )
403 {
404  char *buffer;
405  BYTE printer = 0;
406 
407  if (command != USB_PRINTER_ATTACHED)
408  {
409  // Try to find the current printer. If we can't find the printer in the list,
410  // put it in the list at the first available location.
411  for (printer=0; (printer<USB_MAX_PRINTER_DEVICES) && (printerListPCL[printer].deviceAddress != address); printer++ );
412  if (printer == USB_MAX_PRINTER_DEVICES)
413  {
414  return USB_PRINTER_UNKNOWN_DEVICE;
415  }
416  }
417 
418  // The USB_PRINTER_SET_POSITION should be used only when not using vector graphics.
419  if ((command == USB_PRINTER_SET_POSITION) && (USING_VECTOR_GRAPHICS))
420  {
421  command = USB_PRINTER_GRAPHICS_MOVE_TO;
422  }
423 
424  // If we are connected to a PCL printer that does not support vector graphis,
425  // such as PCL 3, return an error if the command is a vector graphics command.
426  if ((command > USB_PRINTER_VECTOR_GRAPHICS_START) && (command < USB_PRINTER_VECTOR_GRAPHICS_END) && !USING_VECTOR_GRAPHICS)
427  {
428  return USB_PRINTER_UNKNOWN_COMMAND;
429  }
430 
431  switch( command )
432  {
433  //---------------------------------------------------------------------
434  case USB_PRINTER_ATTACHED:
435  for (printer=0; (printer<USB_MAX_PRINTER_DEVICES) && (printerListPCL[printer].deviceAddress != 0); printer++ );
436  if (printer != USB_MAX_PRINTER_DEVICES)
437  {
438  printerListPCL[printer].deviceAddress = address;
439  printerListPCL[printer].support = *((USB_PRINTER_FUNCTION_SUPPORT *)(data.pointerRAM));
440  return USB_PRINTER_SUCCESS;
441  }
442  return USB_PRINTER_TOO_MANY_DEVICES;
443  break;
444 
445  //---------------------------------------------------------------------
446  case USB_PRINTER_DETACHED:
447  for (printer=0; (printer<USB_MAX_PRINTER_DEVICES) && (printerListPCL[printer].deviceAddress != address); printer++ );
448  if (printer != USB_MAX_PRINTER_DEVICES)
449  {
450  printerListPCL[printer].deviceAddress = 0;
451  }
452  return USB_PRINTER_SUCCESS;
453  break;
