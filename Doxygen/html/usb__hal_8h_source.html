500 /**************************************************************************
501  Function:
502  USBHALTransferData
503 
504  Description:
505  This routine prepares to transfer data on the USB.
506  If the system is in device mode, the actual transfer
507  will not occur until the host peforms an OUT request
508  to the given endpoint. If the system is in host mode,
509  the transfer will not start until the token has been
510  sent on the bus.
511 
512  Preconditions:
513  1. USBHALInitialize must have been called to
514  initialize the USB HAL.
515  2. The endpoint through which the data will be
516  transferred must be configured as appropriate by a
517  call to USBHALSetEpConfiguration.
518  3. The bus must have been enumerated (either as a host
519  or device). Except for EP0 transfers.
520 
521  Parameters:
522  flags - Flags consists of the endpoint number OR'd
523  with one or more flags indicating transfer
524  direction and such (see "Data Transfer
525  Macros" in USBCommon.h):
526 
527  7 6 5 4 3 2 1 0 - Description
528  | | | | \_____/
529  | | | | +----- Endpoint Number
530  | | | +---------- Short or zero-size pkt
531  | | +------------ Data Toggle 0/1
532  | +-------------- Force Data Toggle
533  +---------------- 1=Transmit/0=Receive
534 
535  buffer Address of the buffer to receive data.
536 
537  size Number of bytes of data to transfer.
538 
539  Return Values:
540  TRUE if the HAL was able to successfully start the
541  data transfer, FALSE if not.
542 
543  Side Effects:
544  The HAL has prepared to transfer the data on the USB.
545 
546  Ramarks:
547  The HAL will continue the data transfer, keeping track
548  of the buffer address, data remaining, and ping-pong
549  buffer details internally when USBHALHandleBusEvent is
550  called (either polled or in response to an interrupt).
551  The caller will receive notification that the transfer
552  has completed when the EVT_XFER event is passed into
553  the USBHALBusEventCallout call-out function.
554 
555  *************************************************************************/
