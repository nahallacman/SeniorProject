1 /******************************************************************************
2  PCL 5 Printer Language Support
3 
4  Summary:
5  This file provides support for the PCL 5 printer language when using the
6  USB Embedded Host Printer Client Driver.
7 
8  Description:
9  This file provides support for the PCL 5 printer language when using the
10  USB Embedded Host Printer Client Driver.
11 
12  There are several versions of the PCL printer language. This file is
13  targetted to support PCL 5. Unfortunately, printer language support is not
14  always advertised correctly by the printer. Some printers advertise only
15  PCL 6 support when they also support PCL 5. Therefore, the default value
16  for the LANGUAGE_ID_STRING_PCL string used in the routine
17  USBHostPrinterLanguagePCL5IsSupported() is set such that the routine will
18  return TRUE if any PCL language support is advertised. It is highly
19  recommended to test the target application with the specific printer(s)
20  that will be utilized, and, if possible, populate the
21  usbPrinterSpecificLanguage[] array in usb_config.c via the configuration
22  tool to manually select the printer language and its functional support.
23 
24  Notes:
25  The PCL 5 coordinate origin is located at the top left corner of the paper.
26  The HP-GL/2 coordinate origin, however, is located at the bottom left corner
27  of the page. For consistency for the user, HP-GL/2 coordinate system is
28  adjusted to match the PCL coordinate system. This also matches the
29  coordinate system use by the Microchip Graphics library.
30 
31  The black and white bit image polarity is 0=white, 1=black, which is
32  reversed from the Microchip Graphics Library polarity. This driver will
33  automatically convert the image data to the required format, as long as the
34  image data is located in ROM (USB_PRINTER_TRANSFER_FROM_ROM) or it is
35  copied from a RAM buffer (USB_PRINTER_TRANSFER_COPY_DATA). If the data is
36  to be sent directly from its original RAM location, the data must already
37  be in the format required by the printer language.
38 
39  PCL 5 is not compatible with PCL 6; PCL 5 utilizes ASCII input, whereas
40  PCL 6 utilizes binary data. However, some printers that advertise support
41  for only PCL 5 do support PCL 6.
42 
43  PCL 3 printers utilize many of the PCL 5 commands. The following
44  limitations exist with PCL 3:
45  * PCL 3 does not support vector graphics
46  * PCL 3 does not support image printing in landscape mode. The
47  page print will fail.
48  * Items must be sent to the page in the order that they are to be
49  printed from top to bottom. The printer cannot return to a
50  higher position on the page.
51  * PCL 3 does not support the USB_PRINTER_EJECT_PAGE command. Use the
52  USB_PRINTER_JOB_STOP and USB_PRINTER_JOB_START commands instead.
53

119  Function:
120  BYTE USBHostPrinterLanguagePCL5( BYTE address,
121  USB_PRINTER_COMMAND command, USB_DATA_POINTER data, DWORD size, BYTE transferFlags )
122 
123  Summary:
124  This function executes printer commands for a PCL 5 printer.
125 
126  Description:
127  This function executes printer commands for a PCL 5 printer. When
128  the application issues a printer command, the printer client driver
129  determines what language to use to communicate with the printer, and
130  transfers the command to that language support routine. As much as
131  possible, commands are designed to produce the same output regardless
132  of what printer language is used.
133 
134  Not all printer commands support data from both RAM and ROM. Unless
135  otherwise noted, the data pointer is assumed to point to RAM, regardless of
136  the value of transferFlags. Refer to the specific command to see if ROM
137  data is supported.
138 
139  Preconditions:
140  None
141 
142  Parameters:
143  BYTE address - Device's address on the bus
144  USB_PRINTER_COMMAND command - Command to execute. See the enumeration
145  USB_PRINTER_COMMAND for the list of
146  valid commands and their requirements.
147  USB_DATA_POINTER data - Pointer to the required data. Note that
148  the caller must set transferFlags
149  appropriately to indicate if the pointer is
150  a RAM pointer or a ROM pointer.
151  DWORD size - Size of the data. For some commands, this
152  parameter is used to hold the data itself.
153  BYTE transferFlags - Flags that indicate details about the
154  transfer operation. Refer to these flags
155  * USB_PRINTER_TRANSFER_COPY_DATA
156  * USB_PRINTER_TRANSFER_STATIC_DATA
157  * USB_PRINTER_TRANSFER_NOTIFY
158  * USB_PRINTER_TRANSFER_FROM_ROM
159  * USB_PRINTER_TRANSFER_FROM_RAM
160 
161  Return Values:
162  USB_PRINTER_SUCCESS - The command was executed successfully.
163  USB_PRINTER_UNKNOWN_DEVICE - A printer with the indicated address is not
164  attached
165  USB_PRINTER_TOO_MANY_DEVICES - The printer status array does not have
166  space for another printer.
167  USB_PRINTER_OUT_OF_MEMORY - Not enough available heap space to
168  execute the command.
169  other - See possible return codes from the
170  function USBHostPrinterWrite().
171 
172  Remarks:
173  When developing new commands, keep in mind that the function
174  USBHostPrinterCommandReady() will be used before calling this function to
175  see if there is space available in the output transfer queue.
176  USBHostPrinterCommandReady() will routine TRUE if a single space is
177  available in the output queue. Therefore, each command can generate only
178  one output transfer.
179 
180  Multiple printer languages may be used in a single application. The USB
181  Embedded Host Printer Client Driver will call the routine required for the
182  attached device.

216
