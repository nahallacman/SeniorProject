176 /********************************************************************
177  Function:
178  BOOL HIDTxHandleBusy(USB_HANDLE handle)
179 
180  Summary:
181  Retreives the status of the buffer ownership
182 
183  Description:
184  Retreives the status of the buffer ownership. This function will
185  indicate if the previous transfer is complete or not.
186 
187  This function will take the input handle (pointer to a BDT entry) and
188  will check the UOWN bit. If the UOWN bit is set then that indicates
189  that the transfer is not complete and the USB module still owns the data
190  memory. If the UOWN bit is clear that means that the transfer is
191  complete and that the CPU now owns the data memory.
192 
193  For more information about the BDT, please refer to the appropriate
194  datasheet for the device in use.
195 
196  Typical Usage:
197  <code>
198  //make sure that the last transfer isn't busy by checking the handle
199  if(!HIDTxHandleBusy(USBInHandle))
200  {
201  //Send the data contained in the ToSendDataBuffer[] array out on
202  // endpoint HID_EP
203  USBInHandle = HIDTxPacket(HID_EP,(BYTE*)&ToSendDataBuffer[0],sizeof(ToSendDataBuffer));
204  }
205  </code>
206 
207  PreCondition:
208  None.
209 
210  Parameters:
211  USB_HANDLE handle - the handle for the transfer in question.
212  The handle is returned by the HIDTxPacket() and HIDRxPacket()
213  functions. Please insure that USB_HANDLE objects are initialized
214  to NULL.
215 
216  Return Values:
217  TRUE - the HID handle is still busy
218  FALSE - the HID handle is not busy and is ready to send
219  additional data.
220 
221  Remarks:
222  None
