2  ESC/POS Printer Language Support
3 
4  Summary:
5  This file provides support for the ESC/POS printer language when using the
6  USB Embedded Host Printer Client Driver.
7 
8  Description:
9  This file provides support for the ESC/POS printer language when using the
10  USB Embedded Host Printer Client Driver.
11 
12  The exact implementation of ESC/POS varies across manufacturers and even
13  across different models from the same manufacturer. Some POS printers use
14  specialized Device ID strings (obtained with the GET DEVICE ID class-
15  specific request) to indicate the deviations from the strict ESC/POS
16  specification. Also, many printers indicate a custom USB Peripheral Device
17  rather than the Printer Class, and therefore do not support the
18  GET DEVICE ID device request. For example:
19  <code>
20  Printer Tested VID/PID Class Device ID Substring
21  -------------------------------------------------------------
22  Bixolon SRP-270 0x0419/0x3C01 Printer ESC
23  Epson TM-T88IV 0x04B8/0x0202 Custom Not supported
24  Seiko DPU-V445 0x0619/0x0111 Printer Not present
25  Seiko MPU-L465 0x0619/0x0109 Printer SIIMPU
26  </code>
27  Therefore, dynamic language determination is not recommended for POS
28  printers. Instead, create your application to target a either a single
29  printer model or a group of printer models with identical requirements,
30  and indicate specific VID, PID, and printer language via the USB
31  Configuration Tool, and test the application to ensure consistent behavior
32  across all supported printer models.
33 
34  The ESC/POS language support code provides several #defines that allow the
35  language support file to automatically configure itself for different
36  printer models. These #defines can be set using the USB Configuration Tool
37  (USBConfig.exe or MPLAB VDI). Note that they are determined at compile
38  time, not run time, so only one type of printer can be utilized by an
39  application. Printer models other that the ones explicitly tested may
40  require other modifications to the language support code.
41 
42  Notes:
43  Currently, only standard mode is supported.
44 
45  The black and white bit image polarity is 0=white, 1=black, which is
46  reversed from the Microchip Graphics Library polarity. This driver will
47  automatically convert the image data to the required format, as long as the
48  image data is located in ROM (USB_PRINTER_TRANSFER_FROM_ROM) or it is
49  copied from a RAM buffer (USB_PRINTER_TRANSFER_COPY_DATA). If the data is
50  to be sent directly from its original RAM location, the data must already
51  be in the format required by the printer language.

300 /****************************************************************************
301  Function:
302  BYTE USBHostPrinterLanguageESCPOS( BYTE address,
303  USB_PRINTER_COMMAND command, USB_DATA_POINTER data, DWORD size, BYTE transferFlags )
304 
305  Summary:
306  This function executes printer commands for an ESC/POS printer.
307 
308  Description:
309  This function executes printer commands for an ESC/POS printer. When
310  the application issues a printer command, the printer client driver
311  determines what language to use to communicate with the printer, and
312  transfers the command to that language support routine. As much as
313  possible, commands are designed to produce the same output regardless
314  of what printer language is used.
315 
316  Not all printer commands support data from both RAM and ROM. Unless
317  otherwise noted, the data pointer is assumed to point to RAM, regardless of
318  the value of transferFlags. Refer to the specific command to see if ROM
319  data is supported.
320 
321  Preconditions:
322  None
323 
324  Parameters:
325  BYTE address - Device's address on the bus
326  USB_PRINTER_COMMAND command - Command to execute. See the enumeration
327  USB_PRINTER_COMMAND for the list of
328  valid commands and their requirements.
329  USB_DATA_POINTER data - Pointer to the required data. Note that
330  the caller must set transferFlags
331  appropriately to indicate if the pointer is
332  a RAM pointer or a ROM pointer.
333  DWORD size - Size of the data. For some commands, this
334  parameter is used to hold the data itself.
335  BYTE transferFlags - Flags that indicate details about the
336  transfer operation. Refer to these flags
337  * USB_PRINTER_TRANSFER_COPY_DATA
338  * USB_PRINTER_TRANSFER_STATIC_DATA
339  * USB_PRINTER_TRANSFER_NOTIFY
340  * USB_PRINTER_TRANSFER_FROM_ROM
341  * USB_PRINTER_TRANSFER_FROM_RAM
342 
343  Return Values:
344  USB_PRINTER_SUCCESS - The command was executed successfully.
345  USB_PRINTER_UNKNOWN_DEVICE - A printer with the indicated address is not
346  attached
347  USB_PRINTER_TOO_MANY_DEVICES - The printer status array does not have
348  space for another printer.
349  USB_PRINTER_OUT_OF_MEMORY - Not enough available heap space to
350  execute the command.
351  other - See possible return codes from the
352  function USBHostPrinterWrite().
353 
354  Remarks:
355  When developing new commands, keep in mind that the function
356  USBHostPrinterCommandReady() will be used before calling this function to
357  see if there is space available in the output transfer queue.
358  USBHostPrinterCommandReady() will routine TRUE if a single space is
359  available in the output queue. Therefore, each command can generate only
360  one output transfer.
361 
362  Multiple printer languages may be used in a single application. The USB
363  Embedded Host Printer Client Driver will call the routine required for the
364  attached device.
365  ***************************************************************************/

367 BYTE USBHostPrinterLanguageESCPOS( BYTE address,
368  USB_PRINTER_COMMAND command, USB_DATA_POINTER data, DWORD size, BYTE transferFlags )
369 {
370  char *buffer;
371  int i;
372  BYTE printer = 0;
373 
374  if (command != USB_PRINTER_ATTACHED)
375  {
376  // Try to find the current printer. If we can't find the printer in the list,
377  // put it in the list at the first available location.
378  for (printer=0; (printer<USB_MAX_PRINTER_DEVICES) && (printerListESCPOS[printer].deviceAddress != address); printer++ );
379  if (printer == USB_MAX_PRINTER_DEVICES)
380  {
381  return USB_PRINTER_UNKNOWN_DEVICE;
382  }
383  }
384 
385  switch( command )
386  {
387  //---------------------------------------------------------------------
388  case USB_PRINTER_ATTACHED:
389  for (printer=0; (printer<USB_MAX_PRINTER_DEVICES) && (printerListESCPOS[printer].deviceAddress != 0); printer++ );
390  if (printer != USB_MAX_PRINTER_DEVICES)
391  {
392  printerListESCPOS[printer].deviceAddress = address;
393  printerListESCPOS[printer].support = *((USB_PRINTER_FUNCTION_SUPPORT *)(data.pointerRAM));
394  printerListESCPOS[printer].printerFlags.value = 0;
395  return USB_PRINTER_SUCCESS;
396  }
397  return USB_PRINTER_TOO_MANY_DEVICES;
398  break;
399 
400  //---------------------------------------------------------------------
401  case USB_PRINTER_DETACHED:
402  for (printer=0; (printer<USB_MAX_PRINTER_DEVICES) && (printerListESCPOS[printer].deviceAddress != address); printer++ );
403  if (printer != USB_MAX_PRINTER_DEVICES)
404  {
405  printerListESCPOS[printer].deviceAddress = 0;
406  }
407  return USB_PRINTER_SUCCESS;
408  break;
409 
410  //---------------------------------------------------------------------
411  case USB_PRINTER_JOB_START:
412  _SetCurrentPosition( 0, 0 );
413 
414  buffer = (char *)USB_MALLOC( 2 );
415  if (buffer == NULL)
416  {
417  return USB_PRINTER_OUT_OF_MEMORY;
418  }
419 
420  // Initialize printer
421  buffer[0] = ESC_CHAR;
422  buffer[1] = '@';
423 
424  USBHOSTPRINTER_SETFLAG_COPY_DATA( transferFlags );
425  return USBHostPrinterWrite( printerListESCPOS[printer].deviceAddress, buffer, 2, transferFlags );
426  break;
427 
428 
429  //---------------------------------------------------------------------
430  case USB_PRINTER_JOB_STOP:
431  // For compatibility with full sheet operation, if we are in page
432  // mode, perform the USB_PRINTER_EJECT_PAGE command.
433  if (printerListESCPOS[printer].printerFlags.inPageMode)
434  {
435  goto EjectPage;
436  }
437  return USB_PRINTER_SUCCESS;
438  break;
439 
440  //---------------------------------------------------------------------
441  case USB_PRINTER_FONT_NAME:
442  // If an illegal size is specified, return with an error.
443  if (size >= USB_PRINTER_FONT_POS_MAX_FONT)
444  {
445  return USB_PRINTER_BAD_PARAMETER;
446  }
447 
448  switch ((BYTE)size)
449  {
450  case USB_PRINTER_FONT_POS_18x36:
451  size = 0x00;
452  break;
453  case USB_PRINTER_FONT_POS_18x72:
454  size = 0x10;
455  break;
456  case USB_PRINTER_FONT_POS_36x36:
457  size = 0x20;
458  break;
459  case USB_PRINTER_FONT_POS_36x72:
460  size = 0x30;
461  break;
462  case USB_PRINTER_FONT_POS_12x24:
463  size = 0x01;
464  break;
465  case USB_PRINTER_FONT_POS_12x48:
466  size = 0x11;
467  break;
468  case USB_PRINTER_FONT_POS_24x24:
469  size = 0x21;
470  break;
471  case USB_PRINTER_FONT_POS_24x48:
472  size = 0x31;
473  break;
474  }
475 
476  printerListESCPOS[printer].fontName = (BYTE)size;
477  return _PrintFontCommand( printer, transferFlags );
478  break;
479 
480  //---------------------------------------------------------------------
481  case USB_PRINTER_FONT_UPRIGHT:
482  // Italic printing is not supported, but we will not return an
483  // error if the user specifies upright.
484  return USB_PRINTER_SUCCESS;
485  break;

503  if (buffer == NULL)
504  {

506  }

509 
510  USBHOSTPRINTER_SETFLAG_COPY_DATA( transferFlags );
511  return USBHostPrinterWrite( printerListESCPOS[printer].deviceAddress, buffer, 1, transferFlags );
512  break;

516  // No text initialization is required.
517  return USB_PRINTER_SUCCESS;
518  break;
519 
520  //---------------------------------------------------------------------
521  case USB_PRINTER_TEXT:
522  case USB_PRINTER_TRANSPARENT:
523  // If the user's data is in ROM, we have to copy it to RAM first,
524  // so the USB Host routines can read it.

532  if (buffer == NULL)
533  {
534  return USB_PRINTER_OUT_OF_MEMORY;

536

550  }
551  }

567  }

574  buffer = (char *)USB_MALLOC( 3 );
575  if (buffer == NULL)
576  {
577  return USB_PRINTER_OUT_OF_MEMORY;
578  }
579

581  buffer[1] = 'd';
582  buffer[2] = (BYTE)size;
583 
584  USBHOSTPRINTER_SETFLAG_COPY_DATA( transferFlags );
585  return USBHostPrinterWrite( printerListESCPOS[printer].deviceAddress, buffer, 3, transferFlags );
586  break;

589  case USB_PRINTER_IMAGE_START:
590  #define VERTICAL_DENSITY ((USB_PRINTER_IMAGE_INFO *)(data.pointerRAM))->densityVertical
591  #define HORIZONTAL_DENSITY ((USB_PRINTER_IMAGE_INFO *)(data.pointerRAM))->densityHorizontal

597  if (!((VERTICAL_DENSITY == 8) || (VERTICAL_DENSITY == 24)))
598  #else
599  if (!((VERTICAL_DENSITY == 8)))
600  #endif
601  {
602  return USB_PRINTER_BAD_PARAMETER;
603  }

605  {

607  }

626  printerListESCPOS[printer].density = 32;
627  printerListESCPOS[printer].imageDataWidth = 5;
628  }
629  #endif

631  {
632  printerListESCPOS[printer].density ++;
633  }
634

1202  This function formats data for a bitmapped image into the format required
1203  for sending to a POS printer.
1204 
1205  Description:
1206  This function formats data for a bitmapped image into the format required
1207  for sending to a POS printer. Bitmapped images are stored one row of pixels
1208  at a time. Suppose we have an image with vertical black bars, eight pixels
1209  wide and eight pixels deep. The image would appear as the following pixels,
1210  where 0 indicates a black dot and 1 indicates a white dot:
1211  <code>
1212  0 1 0 1 0 1 0 1
1213  0 1 0 1 0 1 0 1
1214  0 1 0 1 0 1 0 1
1215  0 1 0 1 0 1 0 1
1216  0 1 0 1 0 1 0 1
1217  0 1 0 1 0 1 0 1
1218  0 1 0 1 0 1 0 1
1219  0 1 0 1 0 1 0 1
1220  </code>
1221  The stored bitmap of the data would contain the data bytes, where each byte
1222  is one row of data:
1223  <code>
1224  0x55 0x55 0x55 0x55 0x55 0x55 0x55 0x55
1225  </code>
1226  When printing to a full sheet printer, eight separate
1227  USB_PRINTER_IMAGE_DATA_HEADER / USB_PRINTER_IMAGE_DATA command combinations
1228  are required to print this image.
1229 
1230  POS printers, however, require image data formated either 8 dots or 24 dots
1231  deep, depending on the desired (and supported) vertical print density. For
1232  a POS printer performing an 8-dot vertical density print, the data needs to
1233  be in this format:
1234  <code>
1235  0x00 0xFF 0x00 0xFF 0x00 0xFF 0x00 0xFF
1236  </code>

1238  USB_PRINTER_IMAGE_DATA_HEADER / USB_PRINTER_IMAGE_DATA command combination
1239  is required to print this image.
