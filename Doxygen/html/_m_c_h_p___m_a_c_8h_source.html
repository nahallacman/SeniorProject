185  /************************************************************************************
186  * Function:
187  * BOOL MiMAC_SetChannel(BYTE channel, BYTE offsetFreq)
188  *
189  * Summary:
190  * This function set the operating channel for the RF transceiver
191  *
192  * Description:
193  * This is the primary MiMAC interface for the protocol layer to
194  * set the operating frequency of the RF transceiver. Valid channel
195  * number are from 0 to 31. For different frequency band, data rate
196  * and other RF settings, some channels from 0 to 31 might be
197  * unavailable. Paramater offsetFreq is used to fine tune the center
198  * frequency across the frequency band. For transceivers that follow
199  * strict definition of channels, this parameter may be discarded.
200  * The center frequency is calculated as
201  * (LowestFrequency + Channel * ChannelGap + offsetFreq)
202  *
203  * PreCondition:
204  * Hardware initialization on MCU has been done.
205  *
206  * Parameters:
207  * BYTE channel - Channel number. Range from 0 to 31. Not all channels
208  * are available under all conditions.
209  * BYTE offsetFreq - Offset frequency used to fine tune the center
210  * frequency. May not apply to all RF transceivers
211  *
212  * Returns:
213  * A boolean to indicates if channel setting is successful.
214  *
215  * Example:
216  * <code>
217  * // Set center frequency to be exactly channel 12
218  * MiMAC_SetChannel(12, 0);
219  * </code>
220  *
221  * Remarks:
222  * None
223  *
224  *****************************************************************************************/
225  BOOL MiMAC_SetChannel(BYTE channel, BYTE offsetFreq);
