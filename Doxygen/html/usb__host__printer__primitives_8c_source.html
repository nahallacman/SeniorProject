88 /****************************************************************************
89  Function:
90  SHORT PrintScreen( BYTE address, USB_PRINT_SCREEN_INFO *printScreenInfo )
91 
92  Summary:
93  This routine will extract the image that is currently on the specified
94  portion of the graphics display, and print it at the specified location.
95 
96  Description:
97  This routine is intended for use in an application that is using the
98  Graphics Library to control a graphics display. This routine will
99  extract the image that is currently on the specified portion of
100  the graphics display, and print it at the specified location.
101  Since the display may be in color and the printer can print only black
102  and white, the pixel color to interpret as black must be specified in the
103  USB_PRINT_SCREEN_INFO structure.
104 
105  The function can be compiled as either a blocking function or a
106  non-blocking function. When compiled as a blocking function, the routine
107  will wait to enqueue all printer instructions. If an error occurs, then
108  this function will return the error. If all printer instructions are
109  enqueued successfully, the function will return -1. When compiled as a
110  non-blocking function, this function will return 0 if the operation is
111  proceeding correctly but has not yet completed. The application must
112  continue to call this function, with the same parameters, until a
113  non-zero value is returned. A value of -1 indicates that all printer
114  instructions have been enqueued successfully. Any other value is an error
115  code, and the state machine will be set back to the beginning state.
116 
117  Precondition:
118  None
119 
120  Parameters:
121  BYTE address - USB address of the printer.
122  USB_PRINT_SCREEN_INFO *printScreenInfo - Information about the screen
123  area to print, how to interpret the screen image,
124  and how and where to print the image. Note that
125  the width and height members of the structure do
126  not need to be filled in by the application.
127 
128  Return Values:
129  0 - Non-blocking configuration only. Image output is not yet
130  complete, but is proceeding normally.
131  (-1) - Image output was completed successfully.
132  other - Printing was aborted due to an error. See the return values
133  for USBHostPrinterCommand(). Note that the return code
134  USB_PRINTER_SUCCESS will not be returned. Instead, (-1) will
135  be returned upon successful completion.
136 
137  Remarks:
138  None
