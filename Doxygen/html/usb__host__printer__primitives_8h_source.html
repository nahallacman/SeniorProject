104 /****************************************************************************
105  Function:
106  SHORT PrintScreen( BYTE address, USB_PRINT_SCREEN_INFO *printScreenInfo )
107 
108  Summary:
109  This routine will extract the image that is currently on the specified
110  portion of the graphics display, and print it at the specified location.
111 
112  Description:
113  This routine is intended for use in an application that is using the
114  Graphics Library to control a graphics display. This routine will
115  extract the image that is currently on the specified portion of
116  the graphics display, and print it at the specified location.
117  Since the display may be in color and the printer can print only black
118  and white, the pixel color to interpret as black must be specified in the
119  USB_PRINT_SCREEN_INFO structure.
120 
121  The function can be compiled as either a blocking function or a
122  non-blocking function. When compiled as a blocking function, the routine
123  will wait to enqueue all printer instructions. If an error occurs, then
124  this function will return the error. If all printer instructions are
125  enqueued successfully, the function will return -1. When compiled as a
126  non-blocking function, this function will return 0 if the operation is
127  proceeding correctly but has not yet completed. The application must
128  continue to call this function, with the same parameters, until a
129  non-zero value is returned. A value of -1 indicates that all printer
130  instructions have been enqueued successfully. Any other value is an error
131  code, and the state machine will be set back to the beginning state.
132 
133  Precondition:
134  None
135 
136  Parameters:
137  BYTE address - USB address of the printer.
138  USB_PRINT_SCREEN_INFO *printScreenInfo - Information about the screen
139  area to print, how to interpret the screen image,
140  and how and where to print the image. Note that
141  the width and height members of the structure do
142  not need to be filled in by the application.
143 
144  Return Values:
145  0 - Non-blocking configuration only. Image output is not yet
146  complete, but is proceeding normally.
147  (-1) - Image output was completed successfully.
148  other - Printing was aborted due to an error. See the return values
149  for USBHostPrinterCommand(). Note that the return code
150  USB_PRINTER_SUCCESS will not be returned. Instead, (-1) will
151  be returned upon successful completion.
152 
153  Remarks:
154  None
